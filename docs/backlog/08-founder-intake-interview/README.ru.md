## Что это

Вход для founder'а: **минимальная форма заявки** (deck + название компании + опциональные
ссылки на артефакты), за которой следует **чат-интервью с LLM-агентом**, который приходит
предзаполненным из публичных цифровых следов, спрашивает ТОЛЬКО про gaps карточки,
поддерживает **голосовой ввод (ElevenLabs STT) и проигрывание TTS**, и строит карточки
company/founder/team вживую на экране. Плюс **настоящее второе интервью по share-link**,
которое может запросить инвестор (доставка email замокана). Голосовые оригиналы хранятся как
provenance-артефакты.

## Зачем (rubric и обоснование)

- Решение оператора (19 июля): ПОЛНЫЙ цикл, voice in/out в MVP, пометка «Voice agent —
  следующая фаза».
- Боль founder'а №1: время + повторение, не отказ (исследование SeedForge на 4 690 болях:
  37% тянет таймлайн; «одни и те же 15 вопросов 20 раз, нет общей инфраструктуры для proof
  founder'а») — карточка И ЕСТЬ эта инфраструктура; «доказать один раз».
- Минимум по брифу: deck + название компании; избыточный сбор работает против нас (REQ-008,
  FAQ-4). Форма просит артефакты — «GitHub, jupyter NB, replit app — в 100 раз полезнее
  deck'а» (REC-014).
- Guardrails интервью (зафиксировано в roadmap после research): раскрытие заранее + пометка
  о human-review · вопросы только из gaps · pre-fill до того, как спрашивать · ответы = claims
  self-reported (низкий базовый confidence, НЕ оцениваются за красноречие — SIG-018) ·
  результат немедленно · озвученная длительность + progress bar · кнопка «запросить
  человека» · известный риск deterrence смягчён (интервью после формы, а не стена).
- Прецеденты: Parley (FACT-012, 40+ VC — спрос доказан), SeedForge Living Profile; НИ У ОДНОГО
  из 9 OSS-референсов нет интервью, обращённого к founder'у → дифференциация.
- Пилот Zapier Ezra: 97% completion при 4.5/5, если opt-in + human-reviewed; fraud избегает
  завершать интервью → интервью одновременно работает как fraud-фильтр.

## Взгляд на реализацию

n8n + Supabase + фронтенд-компонент чата:

1. **`intake-form`** (фронтенд → Supabase): минимальные поля; создаёт строки
   company+founder+card; триггерит `identity-resolve` + скан pre-fill (саб-workflow'ы фичи
   02).
2. **`interview-agent`** (n8n, webhook на каждое сообщение; промпты через ai-agent-builder):
   контекст = card + список gaps; выбирает следующий вопрос из gap с наивысшей ценностью;
   каждый ответ → claim (source_kind: interview|voice, verbatim сохранён); обновляет
   completeness карточки → live-preview фронтенда перерендеривается (Supabase realtime или
   polling).
3. **Голос**: кнопка микрофона → аудио → n8n webhook → ElevenLabs STT → текст в тот же flow;
   оригинальное аудио → Supabase Storage (`voice_artifacts`); кнопка TTS на каждое сообщение
   агента → ElevenLabs TTS через backend-прокси (ключ никогда не в браузере).
4. **`follow-up-interview`**: инвестор кликает «Request follow-up» на карточке → генерируется
   ссылка share_token (настоящая, открывает интервью, ограниченное выбранными gaps) →
   доставка email ЗАМОКАНА (в UI показывается составленное письмо + ссылка).
5. Финиш: немедленный экран статуса («карточка полная, вердикт в течение 24ч») + ссылка
   opt-out.

## Границы и заглушки

Отправка email замокана. Voice AGENT (слышит интонацию/latency, противодействует ответам с
помощью LLM) — post-MVP со своим отдельным риск-анализом text-vs-voice (parking lot в
roadmap). Без auth для founders — только ссылки share_token.
