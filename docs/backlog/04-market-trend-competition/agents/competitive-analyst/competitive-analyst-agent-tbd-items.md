## Alternatives considered and rejected

| Alternative | Why rejected |
|---|---|
| Let the agent compute the mismatch severity (`minor`/`moderate`/`material`) | §8: "Severity is deterministic, not an LLM judgement", with an explicit four-rung ladder keyed on counts. An LLM severity would be unreproducible and untestable. |
| Let the agent flag founder dishonesty directly | §8 splits the omission into two *different failures with different remedies* — sincerity (trust axis, feature 05) and maturity (`idea_vs_market`, the −10 term) — and REQ-002's whole logic is that collapsing distinct signals hides the disagreement the investor needs. A single "dishonest" flag would collapse them. |
| Default `threat_level` to 3 ("niche") when evidence is thin, instead of null | 3 contributes 0 to §6.4, so it looks equivalent — but it is not: it renders in the memo and the dashboard as an assessed reading rather than an unknown, and it would sort into the competitor table as though it had been established. §3.3's `null` is a first-class value precisely so the absence stays visible. |
| Give the agent a live web-search tool | Breaks the property that makes provenance checkable ("every competitor traces to a document in the input") and bypasses the §2 Tavily credit budget and the `end_date` pinning that makes runs reproducible. |
| Merge the analyst into the sizer as one Sol call | §9 fixes three agents, and the two work different axes with different evidence sets. Merging would also mean the `idea_vs_market` score depends on a call whose primary job is TAM — a failure in one would take out both axes, which REQ-002 exists to prevent. |
| Emit the five buckets as arrays of competitor names alongside `competitors[]` | Duplicate representation drifts. See D7/Q2. |
