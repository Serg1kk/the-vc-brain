# 11 · Demo-данные и слой этики

> English version (primary): [README.md](README.md)

Status: backlog · Depends on: 02-08

## Что это

Субстрат для demo и история про compliance: **3-5 РЕАЛЬНЫХ founders**, найденных нашим
собственным radar (свежий HN Show HN / GitHub) + **1-2 синтетических профиля с seeded
противоречиями** (чтобы безопасно продемонстрировать truth-gap и red flags на сцене), плюс
видимый слой этики: только публичные данные, opt-out, data minimisation, уважение robots.txt.

## Зачем (rubric и обоснование)

- Brief §5: датасет не предоставлен — принеси/синтезируй свой; синтетические профили с
  seeded противоречиями предложены явно; **качество ingestion > размер датасета**.
- Судьи в VC-челлендже спросят про легальность (data-sources.md §legal): план — ответить
  ДО того, как спросят — слайд про этику + видимая механика продукта.
- Реальные founders, найденные вживую = самое сильное доказательство sourcing («его нет на
  Crunchbase — мы его нашли»).
- Решение оператора (19 июля): реальные + 1-2 синтетики.

## Откуда идея

- `internal/research/data-sources.md` — полный юридический/этический план: прецедент hiQ с
  обеих сторон, EDPB 03/2026 + чеклист CNIL, data minimisation как одновременно compliance И
  защита от bias, кнопка «право на возражение», прозрачность источников по каждому score.
- Примеры due-diligence-agents (project-atlas) — форма synthetic-with-contradictions.
- RSK-004: синтетики также позволяют честно показать калибровку, не выставляя слабости
  реального человека на сцене (оптика fairness).

## Взгляд на реализацию

1. **Реальные founders**: прогнать radar (02) на свежем окне HN Show HN во время билда;
   отобрать 3-5 с богатыми цифровыми следами (GitHub-хаб резолвится, краулятся личные
   сайты); прогреть scoring заранее, чтобы demo было мгновенным; хранить raw snapshots как
   provenance.
2. **Синтетики**: 1-2 профиля, собранных вручную через небольшой n8n-генератор или руками:
   один «слишком хорош, чтобы быть правдой» (заявляет 10k пользователей, репозиторий создан
   на прошлой неделе из скопированного источника → флаг provenance + противоречие) и один
   честный cold-start (тонкий цифровой след, высокие confidence-gaps → показывает обработку
   отсутствующих данных). Помечены `synthetic=true` в БД, видимо забейджены в UI (никогда не
   представлены как реальные — честность REQ-004).
3. **Слой этики в продукте**: кнопка opt-out на карточке founder'а + публичная заметка «что
   мы собираем» на странице заявки + data minimisation (нигде нет полей возраста/фото/Art.9)
   + проверка robots.txt, видимая в workflow краулинга. Один слайд в demo-видео: «Только
   публичные данные, официальные API, никаких фейковых аккаунтов, никакого обхода CAPTCHA;
   delete-on-request».

## Границы и заглушки

Без бумажной работы по GDPR (demo, заявлено как есть: «для production в EU —
задокументированная оценка legitimate interest по EDPB 03/2026»). Реальные founders: только
данные профессионального сигнала.
