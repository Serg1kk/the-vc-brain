## What it is

The decision-ready output: a memo with the sponsor's **required sections**, every claim
traced to evidence with confidence, gaps stated honestly, **suggested deep-dive questions
for the founder call**, and a $100K recommendation with conditions. Generated by an n8n
workflow, rendered in the dashboard, exportable.

## Why (rubric & evidence)

- Investment Utility & Execution = 30%: «can a human investor actually act within 24h».
- Required sections (brief Appendix 1 + FAQ-8): Company snapshot, Investment hypotheses,
  SWOT, Problem & product, Traction & KPIs. Everything else optional; **padding counts
  against us** (length ≠ rigor).
- Never fabricate: «Cap table: not disclosed» honesty scores HIGHER trust (REQ-004, FAQ-9).
- Pre-call value: the investor has 30-60 min; memo must say WHERE to dig (REC-005, Carl
  @27:00; PAIN-004).

## Where the idea comes from

- Carl Q&A: recommendation ≠ decision, no founder call in the loop (REQ-001); several scores

- Thesis-template: two blocks — value mechanism + «why now» timing (goes into

## Implementation view

n8n workflow **`generate-memo`** (prompts via ai-agent-builder):

1. Gather: card + all axis scores (latest versions) + claims with statuses + gaps + thesis
   config → context pack.
2. Section writers (one LLM node each, JSON out, citations = claim_ids ONLY — the renderer
   resolves them to sources; a section writer CANNOT introduce an uncited fact):
   snapshot · hypotheses (incl. why-now) · SWOT (each bullet evidence-backed) · problem &
   product · traction & KPIs (verified only; unverified explicitly labeled) · risk matrix ·
   competition (from 04) · financials-lite (benchmark comparables + «not disclosed» marks).
3. **`deep-dive-questions`**: from gaps + contradictions + AMBIGUOUS claims → 5-7 questions
   «to dig deeper on the call», each with WHY (which gap it closes).
4. Decision node (deterministic): thesis fit + axis scores + trust → recommendation
   {proceed / proceed-with-conditions / pass / watchlist} + $100K rationale. Axes NEVER
   averaged — the disagreement between axes is shown as-is.
5. Render: dashboard memo view (09) + markdown export; store versioned in Supabase.

## Boundaries & stubs

Financials/cap table — «not disclosed» + benchmark block only. Exit perspective / DD log —
optional sections, include only if cheap. Adversarial committee pass — stretch (see 05).
