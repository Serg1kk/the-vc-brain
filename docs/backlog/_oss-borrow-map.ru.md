# OSS Borrow-Map — полный отчёт разведчика (19.07)

> English digest (primary): [_oss-borrow-map.md](_oss-borrow-map.md). Ниже — полный отчёт
> агента-разведчика по 9 OSS-репо (пути относительно `internal/other-projects/`).

## 1. vcbrain (TS/pnpm, 5 LLM-агентов → living JSON dossier) — самый богатый по анализам

- **Анализы** — `vcbrain/packages/server/src/skills/*/SKILL.md`. Мега-агент `vc-research/SKILL.md`: 9 суб-анализов — company&identity, **founder deep-dive** («single most important at early stage»), product&moat, **market sizing** (TAM/SAM/SOM + bottoms-up sanity-check + «flag inflated TAMs»), **competitive**, traction+unit-economics (CAC/LTV/NRR/burn-multiple), round&cap-table, signals, diligence-plan. Плюс `vc-thesis-fit` (скоринг vs активной thesis), `vc-memo-writer` (IC-memo, каждый факт cited `[Pitch deck p4]`), `vc-scout` (sourcing YC/PH/GitHub), `vc-portfolio-monitor`.
- **Карточки** — `packages/web/src/pages/Dossier.tsx`: header (category/stage/HQ + fit ScorePill + conviction), Analyst snapshot (thesis/what/**why now**/reco), Thesis fit (bars team/market/product, Fits, Risks, Failed must-haves, **Comps**, **Return scenarios** bear/base/bull), Team+FounderCards, Product, Market, **Competition**, Traction+unit-econ, Round, Diligence plan (red flags+severity), Signals. Pipeline kanban (7 стадий), ThesisStudio (mustHaves/dealBreakers/weights).
- **Данные** — DTO `packages/shared/src/types.ts` (`Startup`, `StartupDossier` ~30 секций, `ThesisConfig`, `ThesisFit`, `Memo`); Drizzle `packages/server/src/schema/{startups,theses,memos,portfolio,files}.ts` (dossier = JSON на строке). `THESIS_DIMENSIONS=[team,market,product]`, веса 34/33/33.
- **Trend/why-now** — `category` free-text; `snapshot.whyNow` + `market.whyNow`/`market.timing` + `tailwinds[]`/`headwinds[]`/`growthRate`/`regulatory`. Численного momentum-скоринга нет — таймингом служит `signals[]` (hiring velocity/funding/launches).
- **Competitive — лучшая структура среди агентных.** `competition = { positioning, moatVsCompetition, direct[], indirect[], substitutes[], barriersToEntry[] }`; direct-запись = `{name, note, funding, vsThem}`. Явного threat-level/switching-cost поля на конкуренте нет.

## 2. vantage (Python/FastAPI/SQLite) — эталон Memory/traceability

- **Анализы** — `vantage/services/scoring.py` + `ai.py`: 7-мерный скоринг (fit .30 / traction .20 / timing .15 / team .15 / novelty .10 / deal_accessibility .10 − risk .15), каждый 0-100 + confidence + evidence. `HeuristicProvider` (офлайн) ↔ `LiveLLMProvider`; **формулу владеет бэкенд** («model proposes, backend decides»). Плюс `ingestion.py`, `entity_resolution.py` (6-шаговый каскад), `memo.py` (bull/bear + PURSUE≥72/WATCH/MONITOR/PASS).
- **Карточки** — Jinja `vantage/templates/`: **Deal Radar** (`radar.html`: score-disc + confidence + **why_now line** + до 4 evidence-chips + «→ Next:» + sector tags), **Company profile** (`company.html`: AI summary, **Why-now card** с trigger-буллетами + «Investor implication», 7 score-bars, **Evidence ledger** +/− с source+signal-id+weight, missing-data), **Memo** (`memo.html`).
- **Данные** — `vantage/models.py`: `Company`(sectors[]), `Signal`(immutable, signal_kind, content_hash dedupe), `SignalCompany`(граф-ребро, relation_type), `Thesis`(target_sectors[]/positive_keywords[]/negative_keywords[]), **`CompanyScore`**(7 суб-скоров + prompt_version + formula_version, versioned), `CompanyMetric`(temporal ledger «velocity is the alpha»), `AIOutput`(anti-black-box ledger). Immutable versioned scores — ключевой паттерн.
- **Trend/why-now — механичнее vcbrain.** `timing_score`+`novelty_score` из `_KIND_WEIGHTS` по signal-kind; **momentum-бонус** (≥2 hiring → timing+8/traction+6, `ai.py:118-124`); temporal ledger = структурный «why-now over time». `_why_now()` (`ai.py:265-279`) генерит строку: «{n} recent hiring signals indicate a go-to-market scaling window», «recent financing confirms momentum», «squarely inside the '{sector}' target sector», иначе «No acute timing catalyst detected yet; worth watching».
- **Competitive — НЕТ.** Ни дискавери, ни competitor-сущности, ни differentiation/threat/moat полей. Самый явный gap репо.

## 3. reporting (Next.js/Supabase, 6-стадийный memo-агент) — эталон competitive + provenance

- **Анализы** — email-routing (`lib/pipeline/classifyEmail.ts`), **deal-screening/thesis-fit** (`lib/claude/analyzeDeal.ts` → thesis_fit_score strong/moderate/weak/out_of_thesis/spam), external deal-research (`lib/deals/research.ts`, gated по fit-бару), 6-стадийный memo-агент `lib/memo-agent/stages/*` (ingest→research→qa→draft→score→render) + `checklist-assessment.ts`.
- **Карточки** — `components/diligence/research-summary.tsx`: stat-tiles (Findings/Verified/Contradicted/Gaps), Contradictions (material/minor), Findings (verification-badge + tiered sources), **Competitive map** (2 колонки: *Named by company* {name,note} | *Identified by research* {name,rationale}), Founder dossiers, Research gaps. Memo — `lib/memo-agent/defaults/memo_output.yaml`: header, exec_summary, recommendation (partner-only), overview, product, market, traction, team, **competition_moat**, outcomes_analysis (return scenarios), risks, scoring_summary; каждый абзац = `paragraph_record{sources[], contains_projection/unverified/contradiction}`.
- **Данные** — `supabase/migrations/*`: `inbound_deals`(thesis_fit_analysis, research_findings jsonb), `diligence_deals`(current_memo_stage), `diligence_memo_drafts`(стадии jsonb), `diligence_checklist_items`(found/partial/missing/na), **`firm_schemas`**(per-fund YAML: rubric/qa_library/research_dossier/memo_output). Config-as-data: `lib/memo-agent/defaults/*.yaml`.
- **Trend — слабо.** Нет движка категоризации/тайминга; «why now» — только опциональная firm-идиосинкразия; market rubric dim (score 5 = «structural shift creating a new category», сигналы: regulatory tailwinds / recent funding).
- **Competitive — ЛУЧШИЙ ОБРАЗЕЦ.** `lib/memo-agent/defaults/research_dossier.yaml`: *«most valuable output is usually the competitors the company did NOT mention»*. Выводы: company_named_competitors, **company_unnamed_competitors** (highest-value), adjacent_competitors, incumbent_competitors, **build_vs_buy_alternatives** (*«Sometimes 'a spreadsheet' is the real competitor»*). **`per_competitor_record`** = `{id, name, category[direct|adjacent|incumbent|alternative], company_mentioned bool, positioning_summary, stage, most_recent_funding{round,amount,date,lead}, differentiation_vs_target, source_urls}`. threat/moat/switching-costs — только в `competition_moat` rubric-dim (не на записи конкурента). Дискавери: regex из ingestion + веб-поиск для неназванных.

## 4. InGa (CLI-first, FalkorDB граф, 10 Claude-агентов) — Memory-граф + EU-data sourcing

- **Анализы** — `vault-template/99_config/agent-roles/*.md`: thesis-parser, discovery (EPO/CORDIS/TED/Offeneregister/Wikidata), entity-resolver, qualifier (**qualify-score 0-100** = CPC 40 + geo 20 + stage 20 + keyword 20; **strategic-fit 0-10** = Claude vs thesis), dossier-writer, path-finder (warm-intro graph shortestPath ≤4 hops), outreach-composer. Сорсинг+outreach, НЕ diligence/memo.
- **Карточки** — dossier `dossier-writer.md` (8 секций): TL;DR, Company Snapshot(ARR/NRR), Founders(+patents), IP&Funding (patents+grants EXIST/BMBF/CORDIS), **Graph Context**(hops/shared investors), Strategic Fit (per-criterion PASS/STRONG table), **Red Flags**, Warm-Intro Paths, Draft Messages.
- **Данные** — FalkorDB `src/lib/graph/schema.ts` (немецкая доменная схема): **6 node-типов** (Startup{employee_growth_6m/12m}, Patent{cpc_klasse}, Person, Foerderung/grant, Forschungsinstitut, Auftrag/tender), **7 edge-типов** все с `{confidence, match_method}` (HAT_PATENT, SPINOFF, ERHIELT, GRUENDETE, GEWANN_AUFTRAG…). Drizzle/Turso `src/lib/db/schema.ts` (+ `intelligence`: category icp|channel|**competitive**|timing, confidence-scored, hypothesis→validated→proven). ⚠️ docs (`docs/agents.md`) описывают английскую схему Company/Founder — рассинхрон с кодом.
- **Trend/why-now — нет движка.** Ближайшее: recency коннекторов (свежие патенты/гранты/тендеры per CPC = momentum-прокси, не скорится); «why now» пишет человек в thesis.md (напр. DORA/EU-AI-Act).
- **Competitive — нет структурной сущности.** Только `intelligence.category='competitive'` (learned GTM) + неявная граф-плотность (same-CPC патенты / same-CPV тендеры). В dossier — прозой в Red Flags/Strategic Fit.

## 5. sieve-mcp — таксономия + evidence-typing (код за закрытым SaaS)

- ⚠️ Тонкий MCP-клиент, вся логика за закрытым SaaS-API (`api.sieve.arceusxventures.com`). Промптов/скоринга в репо нет.
- **IMPACT-X 7×0-20=0-140** (`src/sieve_mcp/server.py:40-47`): **I**nnovators(team), **M**arket(size/timing/tailwinds), **P**roduct(differentiation), **A**dvantage(«competitive moat, defensibility, switching costs»), **C**ommerce(unit-econ), **T**raction, **X**-Factor. Red-flag detection («founder risk, market timing, unsustainable unit-econ, missing traction, competitive threats»), **Evidence-typing Documented/Discovered/Inferred/Missing**, Sector-Aware, Stage-Calibrated.
- `openapi-spec.yaml` `AnalysisSummaryResponse`: `sieve_score`(total+per-dim), **meeting_decision**(Take/Pass/Need-More-Info), executive_summary, key_strengths[], key_concerns[]. Memo internal(IC)/external(founder). Score-bands 100+/70-99/<70. **Ценность = таксономия + словари + decision-labels + memo-split, не код.**

## 6. company-research-agent (LangGraph + Tavily, 2k★) — рабочий enrichment-движок

- **Pipeline** `backend/graph.py`: grounding → 4 параллельных researcher → collector → curator (relevance ≥0.4) → enricher → briefing → editor. Промпты researcher'ов в `backend/prompts.py`: **Company** (fundamentals), **Financial** (funding/valuation/metrics), **Industry** («Market position / Competitors / {industry} trends and challenges / Market size and growth»), **News**. Каждый — max 4 Tavily-запроса.
- **Отчёт**: секции Company (Core Product/Leadership/Target Market/Differentiators/Business Model), **Industry** (Market Overview, **Direct Competition**, Competitive Advantages, Market Challenges), Financial, News; жёсткие хедеры, References MLA top-10 по Tavily score. `backend/classes/state.py` `ResearchState`: per-category raw+curated buckets.
- **Trend** — в Industry researcher (Market Overview: сегмент + size + growth rate). Нет why-now; `industry` = user-input.
- **Competitive** — free-text буллеты `### Direct Competition` + `### Competitive Advantages`. Структурной сущности нет. **Ценность = borrowable-промпты + fan-out/curate/brief/edit механика → портировать в n8n.**

## 7. venture-capital-intelligence (Claude-плагин, 9 скиллов) — эталон market-size + рубрик

- **Анализы** (паттерн: Claude извлекает → Python считает → Claude интерпретирует): soft/hard-screening (**8-dim**: Team .25/Market .20/Product .15/Traction .15/BizModel .10/**Competition .08**/Financials .05/Risk .02; PASS≥7.5), analyze-pitch-deck (11 слайдов + **10 red-flag детекторов**: TAM-abuse, Fake-2×2, hockey-stick), **deal-sourcing-signals** (6-signal Hiring/Funding/Product/Team/Market/Tech, `signal_scorer.py` → MONITOR/ENGAGE/MOVE-FAST), **market-size** (TAM/SAM/SOM + competitors), cap-table-waterfall (OCF), financial-model (DCF/comps/SaaS-health), fund-operations, explain-equity-terms.
- **Данные** — `hard-screening` `company_profile.json`: `{scores{8 dims→{score,rationale}}, investment_thesis, why_now, key_risks[], dd_priorities[], comparables[]}`. `deal-sourcing` `raw_signals.json`: `signals[{type,description,source,date,strength,sentiment}]`.
- **Trend/why-now — сильнейший численный образец.** `market-size` + `scripts/tam_calculator.py`: `VENTURE_TAM_THRESHOLD=$1B`, `GOOD_CAGR_THRESHOLD=0.15`, 5-yr projection `tam*(1+cagr)**5`, `vc_rule_check()` → PASS/WATCH/FAIL. `top_down{total_market_size_usd, addressable_fraction, obtainable_fraction, cagr_pct, source}`. «Why now» — first-class поле.
- **Competitive — типизированная сущность.** `market_inputs.competitors[]` = **`{name, funding_total_usd, estimated_arr_usd, founded_year, differentiation}`** + per-competitor **tech-stack** (webappanalyzer: frontend/backend/db/infra → «technical maturity, rebuild risk, hiring difficulty»). Дискавери: `'[company] competitors' OR '[category] startups' funding 2024`.

## 8. dealflow (Claude-плагин, 15 модулей PE/VC lifecycle) — эталон firmstyle-конфига + deskresearch

- **Анализы** — prescreen (buy-box fit + thesis/anti-thesis + PASS/PURSUE…), **deskresearch** (market sizing/competitive/customer/regulatory/hiring; 3 параллельных субагента; tiered sources Tier-1/2/3), dataroom (rubric + 🟢🟡🔴), model, questions, cohort (NRR/GRR/HHI), superanalyst, returns (LBO/IRR/MOIC), vp-review, pre-ic (devil's advocate + top-10 IC-questions), termsheet, process.
- **Данные** — `config/defaults/{venture-capital,…}.yaml`: `rubric.categories[]{name,weight,questions[]}` (VC: Team/Market-TAM/Product-PMF/Traction/BizModel/Burn/**Competition**/Legal-Cap-Table), `buy_box`, `preferences`. **`firm-style.yaml`** (`scripts/dealflow_lib/firmstyle.py`) = де-факто thesis/voice движок: voice + templates + `term_preferences` + prescreen_config. Deal-state `<deal>/.dealflow/deal-state.yaml` + `index.jsonl`.
- **Trend** — deskresearch market-context: «Industry size/growth/structure · Trends shaping next 3-5 years · Regulatory · Sources: trade press, SEC EDGAR, USPTO». «Why now» — в VC-rubric Market questions.
- **Competitive** — deskresearch субагент: «Direct + adjacent · Recent funding/M&A · Positioning differences · Sources: Crunchbase, press, product pages», каждый claim с цитатой Tier-1/2/3. Типизированной записи нет.

## 9. Deal_flow_analyzer (CrewAI + FastAPI, 6 агентов) — единственный dedicated competitive-агент с web-discovery

- **Анализы** — `backend/app/crew/agents.py`+`tasks.py`, sequential: (1) document_extractor, (2) business_analyst (problem/solution/market_size/revenue_model/traction), (3) team_evaluator (Founder×40+Complete×30+Exp×30), (4) financial_analyst (CAC/LTV/burn/runway + stage-benchmarks), (5) **competitive_analyst**, (6) deal_scorer (Team 35/Business 30/Traction 20/Fin 10/Comp 5; STRONG PASS 85+/PASS/WEAK/CONDITIONAL/FAIL).
- **Карточки/данные** — `frontend/app.js`: overall-score + reco-badge + agent-cards. **Competitive card**: Competitors(tags) + Competitive Advantages + **Moat Assessment**. DB `backend/app/models.py` (JSONB на агента). Pydantic `schemas.py`.
- **Competitive — dedicated агент с активным web-discovery.** `analyze_competition_task` → **`{competitors[], competitive_advantages[], differentiation_strength, moat, market_position, score, concerns[], web_research{search_queries, competitors_found, market_insights}}`**. Активный поиск `'[industry] top competitors 2024', '[company] vs [competitor]'` — находит неназванных в деке. Bands: «85-100 Strong moat/defensible … 30-49 Crowded market, weak differentiation». Вес в финале только 5%.
