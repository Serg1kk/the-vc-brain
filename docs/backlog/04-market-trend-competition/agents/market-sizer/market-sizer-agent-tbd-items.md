# `market-sizer` — Decisions & Open Items

## Decisions taken (with rationale)

| # | Decision | Rationale | Source |
|---|---|---|---|
| D1 | The agent's admissible world is `documents[]` only; its own training-data recall of market figures is explicitly inadmissible | §1's core finding: fabricated report-mill numbers propagate into LLM training data and come back as confident sourceless answers. An agent permitted to "remember" a market size reintroduces exactly the laundering the feature is built against. | design §1, §9 |
| D2 | `size_bottom_up`, `size_top_down`, `growth`, `why_now` are emitted as objects matching §3.2's `value` shapes **field for field**, written verbatim as `claims.value` | Any remap in the write node is where field drift enters. One-to-one is checkable by eye. | design §3.2 |
| D3 | Verbatim quotes live in a **separate top-level `evidence[]` array**, not inline in the size records | §3.2's claim `value` shapes are fixed and contain no quote field; §3.2 states explicitly that for derived rows the word-for-word sentence lives in `evidence.quote_verbatim`. A separate array maps 1:1 onto evidence rows. | design §3.2, §3.5 |
| D4 | `evidence[].supports_field` is a constrained dotted path, enforced by regex in the schema | The write node must attach each evidence row to the right claim; a free-text field would make that attachment an LLM judgement. The regex makes it impossible to cite a field that was not populated. | design §3.5 |
| D5 | Abstention carries `status.ok = true` | §9 makes abstention a valid answer producing a `missing` claim; §4 routes it to the gap writer, not the error branch. `ok: false` would put honesty on the failure path. | design §4, §9 |
| D6 | The incumbent-sizing prohibition is stated in the prompt at full length with worked negative examples, **and** enforced by the validator | §6.4 says "a hard rule in both the prompt and the validator". Belt and braces: the prompt makes the model abstain, the validator catches it if it does not. | design §6.4 |
| D7 | `gaps[].reason_code` is a closed enum of ten values | The validator maps gaps onto `missing_flags` keys deterministically (§3.7) and `missing_count` feeds the confidence formula (§6.5). Free-text reasons would make that mapping a second LLM judgement. | design §3.7, §6.5 |
| D8 | The agent emits **no** score, band, verdict, implied-exit value, outlook label or trend adjective | §6's opening rule: "the LLM never emits the axis number… a deterministic Code node owns the formula". `trend` in particular is a news histogram (§5), never an adjective. | design §5, §6 |
| D9 | `documents[].content` truncated to 6,000 chars, payload capped at 20 documents | Guards the 272K-token pricing cliff (2× input / 1.5× output on the whole request). Anchors sit near the tops of the pages that carry them. | `internal/research/openai/02-models.md`; design §4 (`/extract` ≤ 20 URLs) |
| D10 | Evidence **tier is not passed in** and the agent never emits one; it is asked only to grade source *class* in prose | §3.4: the domain→tier map is "a constant in `f04-config`, not prompt guidance". Letting the model assign tiers would move the anti-report-mill enforcement from data into a prompt, which is precisely what §3.4 refuses. | design §3.4 |
| D11 | `searches_performed[]` is passed in | §3.5 requires `tier='missing'` evidence rows with `relation='context'` recording that a search ran and returned nothing, discriminated by `query`. The agent cannot distinguish "searched, empty" from "never searched" without it. | design §3.5 |
| D12 | Non-English `quote_verbatim` stays in the original language, unedited | The verbatim layer exists to defend against the model's own re-centering (RSK-003); translating or tidying the sentence removes the defence. | design §2 (RSK-003) |
| D13 | `reasoning_effort = high` for the demo, `medium` for bulk | Cost is real here: ~$0.43/call at `high`, ~$43 for 100 cards against a shared $50 pool. Documented so it is a decision, not a surprise. | `internal/research/openai/02-models.md` |

| **Q4** | The `documents[]` payload includes Q3/Q4 competitor-discovery results, and §4 runs the competition sub-call **before** the sizer — so competitor records exist by the time this agent runs. Competitor pricing pages already reach the sizer through `documents[]` (Q2 pricing query + curator), which covers the useful case. Raise if D1 (integration run) shows ARPU anchors going missing. |

Whether the agent should scale the figure down is unspecified — and scaling would be an unsourced derivation. | Reading adopted: **do not scale.** Emit the figure with the mismatch declared in `assumptions[]` and a `geography_mismatch` gap, or abstain. ## Alternatives considered and rejected

| Alternative | Why rejected |
|---|---|
| Give the sizer a live web-search tool (Responses API `web_search`) | Breaks the property that makes provenance checkable: "every number traces to a document in the input". With a search tool, the input no longer bounds the output, and design §2 has already fixed the Tavily budget and the `end_date` pinning that makes runs reproducible. |
| Let the agent compute the TAM band / venture-scale check itself | §6: "model proposes, backend decides"; the formula lives in a Code node with a stored `formula_version`. An LLM-computed band cannot be unit-tested, which is the whole premise of the plan's Stage B. |
| Have the agent output a single triangulated point estimate with a confidence interval | §9: "ranges, never point estimates — a single TAM number is a red flag". A confidence interval computed by an LLM over non-independent sources is a fabricated statistic wearing statistical clothing. |
| Average conflicting sources and record the variance separately | Directly violates §9's "triangulate, do not average". The mean of two contradictory claims appears in no document and is a third, sourceless claim. |
| Second-model check on the size claim (populating `ai_runs.disagreement`) | Design §12 already lists this as a stretch item, gated on the QA gate clearing early. Not in scope for A2; noted so it is not lost. |
| Pass evidence tiers into the prompt so the agent can weight sources itself | §3.4 puts the domain→tier map in config precisely so the anti-report-mill guarantee is data, not prompt guidance. Moving it into the prompt would make the default-deny rule (unknown domain → `inferred`) an LLM judgement. |
