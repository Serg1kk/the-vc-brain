## Проблема

Лучшие фаундеры могут никогда не получить финансирование — не потому что идея слабая, а потому что никто с чековой книжкой не знает их имени. Их история разбросана по питч-декам, GitHub-репозиториям, полусобранным сайтам и постам, которые никто внимательно не читает. Классический диллидженс — это недели звонков, референсов и ужинов: экономика, которая не работает для чеков $100K. **Капитал течёт по нетворкам, а не по merit.** А ранние команды просто невидимы для существующих VC-инструментов: их ещё нет ни в Crunchbase, ни в Dealroom.

## О челлендже

Публичный бриф спонсора просит data- и AI-first операционную систему, меняющую работу венчура — *«крупнейший в мире Shark Tank для AI-инноваций»*:

- **Пайплайн в скоупе:** `Sourcing → Screening → Diligence → Decision`. Downstream-стадии (портфель, follow-on, fund ops, exit) — явно вне скоупа.
- **Три столпа:** **Sourcing** (находить исключительных фаундеров до формального фандрейзинга — важнейшая часть), **Assessment & Intelligence** (reasoning-слой: прозрачен про confidence, неопределённость и evidence), **Memory** (фундамент данных: дедупликация, timestamps, source-теги; ничего не выбрасывается).
- **Founder Score** — «кредитный скор для фаундеров»: живой evidence-backed профиль скиллов и track record, который персистит между заявками и обостряется с каждым майлстоуном.
- **Два трека входа:** фаундер подаёт заявку через минимальную форму (дек + название компании), либо система сама находит его по сигналам (GitHub, лончи, хакатоны, статьи, когорты акселераторов) — оба трека сходятся в одну воронку скрининга.
- **Мульти-осевой скрининг:** Founder / Market / Idea-vs-Market скорятся **независимо** (никогда не усредняются в одно число), каждая ось — со своим трендом.
- **Evidence-backed memo:** каждый клейм — traction, выручка, бэкграунд команды, размер рынка — трейсится к evidence с уровнем уверенности (Trust Score). Отсутствующие данные честно помечаются, а не выдумываются.
- **Выход:** decision-ready инвест-мемо + рекомендация по $100K, по которой инвестор-человек может действовать в течение 24 часов.

Рубрика жюри: Data Architecture & Intelligence (30%) · Investment Utility & Execution (30%) · Intelligent Analysis & Trust (25%) · User Experience & Design (15%). Самая сложная и самая ценная задача — **cold-start фаундер** без track record.

## Что мы строим

Рабочий прототип этого пайплайна с осознанным фокусом на самом важном:

- **поиск и скоринг фаундеров по первичным публичным следам** — до того, как фаундер вообще выйдет на фандрейзинг,
- **per-claim trust scores** — каждое утверждение привязано к evidence; пробелы снижают уверенность, а не оценку фаундера,
- **независимые оси скоринга с трендами** — расхождение осей показывается, а не прячется,
- **decision-ready memo** — честные про то, что система знает и чего не знает,
- **API-first дизайн** — один сервис работает и для людей (веб-UI), и для агентов (API / CLI / MCP).
