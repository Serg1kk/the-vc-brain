# 07 · Thesis Engine (настраиваемая линза фонда)

> English version (primary): [README.md](README.md)

Status: backlog · Depends on: 01

## Что это

Инвестиционный тезис фонда как настраиваемый объект: секторы, стадия, география, размер
чека, целевая доля владения, аппетит к риску. Работает в трёх местах: (a) **pre-filter
gate** на intake (дешёвая первая дверь), (b) **линза на feed** (сортировка/фильтрация по
thesis-fit), (c) **контекст для memo** (рекомендация относительна thesis).

## Зачем (rubric и обоснование)

- MVP-обязательное по брифу §Must-demonstrate 1: «каждая рекомендация отфильтрована и
  оценена через линзу конкретного фонда»; хардкод убивает этот столп (FAQ-15 — явно
  настраиваемо).
- Открытая дверь + pre-filter (REC-003, Carl @47:30): «первая дверь и вторая дверь… нам не
  интересно финансировать очередную ice cream shop». Экономика двухступенчатого screening
  (REC-008): дешёвый текстовый gate → дорогой полный анализ только после gate.
- Их собственный пример: фильтр B2B tech, рейтинг 1-5, 4-5 передаются людям (FACT-001/002).

## Откуда идея

- vcbrain **Thesis Studio** (MIT): must-haves / deal-breakers / взвешенные измерения —
  лучшая OSS-форма этого; адаптировать схему, не код.
- Концепция конфига `firmstyle` из dealflow; текст Sequoia «Services as Software», хранится
  в KB как пример документа thesis для демонстрации thesis-matching.

## Взгляд на реализацию

- Таблица Supabase `thesis`: одна строка на конфиг фонда {sectors[], stages[], geos[],
  check_range, ownership_target, risk_appetite, must_haves[], deal_breakers[], weights_json,
  version}.
- UI (09): простая форма конфига, 6-8 полей + редактируемые чипы; смена thesis
  пересортировывает feed вживую (это wow-момент «линзы» для демо).
- n8n: саб-workflow **`thesis-gate`** (дешёвая LLM-классификация: попадает в thesis?
  сработал deal-breaker? → pass/soft-fail с причиной — soft-fail всё равно сохраняется,
  показывается под «Outside thesis», чтобы открытая дверь оставалась видимой);
  **`thesis-fit-score`** используется ранжированием feed и memo.
- Рекомендация в memo цитирует, какие правила thesis сработали.

## Границы и заглушки

Один фонд (без multi-tenancy). Back-testing thesis (есть у vcbrain) — пометка post-MVP.
